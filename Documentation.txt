Phase 1: HTML Structure Foundation
File: index.html
1. Container Setup
   - Created main `.container` div for all content
   - Added semantic structure: header, main, footer
2. Header Section
   - Title with fire icon
   - Theme toggle button (initially "Dark Mode")
   - Next Day button for progression
3. Calendar Section
   - GitHub-style calendar container
   - Month columns placeholder (JS will populate)
   - Activity legend with intensity colors
   - Current date and total streak display
4. Habits Section
   - Habits container (JS will populate cards)
   - Each habit will have: checkbox, name, streak, progress bar
5. Footer
   - Instructions about forward-only progression
   - Current day counter
--------------------------------------------------------------------------------------------------------------------------------------
Phase 2: CSS Styling & Theming
File: styles.css
1. CSS Variables Setup
   - Created light theme variables in :root
   - Created dark theme overrides in [data-theme="dark"]
   - Defined 5 unique habit colors
2. Responsive Layout
   - Mobile-first approach with media queries
   - Flexbox for calendar and habit cards
   - Grid for calendar day cells
3. Component Styling
   - Calendar: GitHub-style day cells with intensity colors
   - Habit cards: Hover effects, progress bars with habit colors
   - Buttons: Interactive states with transform effects
4. Theme Switching
   - data-theme attribute controls all colors
   - Smooth transitions between themes
--------------------------------------------------------------------------------------------------------------------------------------
Phase 3: JavaScript Core Functionality
File: script.js
1. Initialization (initApp function)
   - Set up event listeners
   - Initialize default habits
   - Build calendar structure
   - Render initial state
2. State Management
   - currentDate: Tracks current day
   - habits[]: Array of habit objects
   - calendarData{}: Maps dates to completion data
   - currentDayIndex: Day counter
3. Theme Toggle (setupThemeToggle function)
   - Checks localStorage for saved preference
   - Checks system preference
   - Switches between light/dark themes
   - Updates button icon/text
4. Calendar System (initializeCalendar function)
   - Creates 12 month columns
   - Calculates day grids with proper offset
   - Adds day cells with data-date attributes
5. Habit Management (renderHabits function)
   - Creates habit cards dynamically
   - Calculates progress percentages
   - Updates streak displays
   - Connects checkboxes to toggleHabitCompletion
6. Calendar Rendering (renderCalendar function)
   - Colors cells based on completion intensity
   - Adds tooltips with completion info
   - Highlights current day
   - Uses opacity for intensity levels
7. Date Progression (goToNextDay function)
   - Validates before moving forward
   - Resets daily completion status
   - Saves data to localStorage
   - Updates all displays

Data Structure Design
Habit Object:
javascript
{
  id: number,
  name: string,
  colorClass: string, // CSS class for color
  streak: number, // Current consecutive days
  completedToday: boolean,
  history: {} // Optional: past completion data
}
Calendar Data Structure:
javascript
calendarData = {
  "2025-01-01": {
    date: Date object,
    habits: {
      1: true, // Habit ID: completion status
      2: false
    }
  }
}

Technical Decisions & Notes
1. Forward-Only Progression
Intentionally no "back" button to maintain streak integrity
Confirmation dialog when moving forward with no habits completed

2. Color System
Each habit has unique color defined in CSS variables
Calendar intensity uses opacity levels on habit colors
Theme switching adjusts all colors automatically

3. Date Handling
Fixed to year 2025 for consistency
Uses YYYY-MM-DD format as object keys
Day cells store dates as data attributes

4. State Management
Simple object/array system (no external libraries)
localStorage for persistence
Centralized rendering functions update all UI components

5. Performance Considerations
Batch DOM updates in render functions
Event delegation could be added for many habits
localStorage operations are minimal
